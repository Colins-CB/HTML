# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.1.x   | :white_check_mark: |
| 2.0.x   | :x:                |
| 1.5.x   | :white_check_mark: |
| < 1.5   | :x:                |

## Reporting a Vulnerability

We take the security of our project seriously. If you believe you have found a security vulnerability, please follow these steps:

1. **Do Not Open a Public Issue:** To protect our users, please do not report security bugs through public GitHub issues.
2. **Email Us:** Send a detailed report to **colinsbett94@gmail.com**. 
3. **Information to Include:**
    * A description of the vulnerability.
    * Steps to reproduce the issue (a Proof of Concept).
    * Potential impact if exploited.

### Our Process
* **Acknowledgment:** You will receive an automated receipt immediately and a personal response within **3 business days**.
* **Fixing:** We aim to release a patch for critical vulnerabilities within **10 days**.
* **Credit:** We are happy to credit you in our release notes for your responsible disclosure!

---
*Please refrain from sharing information about the vulnerability with the public until we have had the opportunity to address it.*
